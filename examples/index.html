The floating toolbar is a hard-coded demo component (examples/toolbar.js) - it is not part of the published package. Choose a selector type below to see how each one is built from the selection by rangeToSelectors() and serialized to RDFa and W3C Web Annotation JSON-LD. The default auto builds every applicable selector at once (https://www.w3.org/TR/annotation-model/#selectors : co-equal alternatives, a consumer picks one). This demo stores a single selector, so under auto it marks and serializes with the first and lists the rest in the object panel.

Web Annotations

The W3C Web Annotation Data Model provides a standard way to express annotations about resources on the web. Annotations can target specific passages of text using selectors such as the TextQuoteSelector.

A TextQuoteSelector identifies a passage by its exact text content, along with a short prefix and suffix for disambiguation when the same phrase appears multiple times.

  • foo
  • bar
  • baz

Annotations can be stored in personal data pods, ActivityStreams outboxes, or external annotation services - and retrieved later to reconstruct the highlighted passages.
